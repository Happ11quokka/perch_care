# 댓글 응대 템플릿

**발행 후 48시간 댓글 실시간 대응 권장.**
2월 포스트 경험상 초기 참여 속도가 노출 알고리즘에 결정적 영향을 줍니다.

---

## 공통 원칙

- **1시간 이내 첫 답변** 목표 (특히 발행 직후 2~4시간)
- 부정적 피드백에도 방어적 반응 금지, "해결하겠다"는 구체적 액션 약속
- 동일 질문이라도 템플릿 복붙 X — 최소 한 문장씩 이름 언급/맥락 반영해 맞춤화
- 이모지 1개 이내, 과용 금지 (진정성 톤 유지)
- 기술 용어 (API, 서버, 업데이트 버전 등) 대신 사용자 언어로 답변

## ② 버그 제보 응대

**변형 — 재현 안 되는 버그:**

```
谢谢告诉我们！这个问题我们想仔细查清楚。
能私信一下您的机型、系统版本，
以及出问题时您刚做了什么操作吗？
这样我们才能一步步复现修复 🙏
```

## ④ 기능 요청 응대

**상황:** "能不能加 XXX 功能", "希望有 YYY", "如果能 ZZZ 就好了" 등

```
这个建议非常好，已经记下来了！🦜
下个版本开发时会优先考虑。
也欢迎继续告诉我们您的想法～
```

**변형 — 구체적/실현 가능한 요청:**
```
太棒的建议了！这个其实我们也在考虑 ✨
能私信详细说说您希望它怎么用吗？
如果有具体的使用场景，
我们就能做得更贴合大家的实际需要。
```

**변형 — 이미 있는 기능:**
```
其实 2.0 里已经可以做这件事啦！🦜
在 [具体位置] 就能找到，试试看～
如果找不到或者用起来不方便，随时告诉我们。
```

**⚠️ 특수 케이스 — "能不能查询鹦鹉能不能吃这个食物?" 류 식품 안전 문의:**

> 본 기능은 `feat/chatbot-multimodal` 브랜치 개발 중, **v2.0에 미배포**. 2월 댓글에서 같은 요청을 받았지만 이번 릴리스엔 없음. 절대 "이미 있다" 응답 금지.

```
非常感谢这个建议！🦜
其实已经在开发中了，不过这次 2.0 还没来得及放进来 🙏
下一版我们会重点做这个功能，
到时候第一时间告诉您～
```

**⚠️ 특수 케이스 — "健康报告链接"이 안 열린다/만료됐다:**

```
感谢反馈！健康报告链接默认 7 天有效，
过期的话在 app 里重新生成一个新的就可以啦～
如果是 7 天内打不开，麻烦私信一下具体情况，
我们立刻查 🙏
```

## ⚠️ 대응 금기 사항

- ❌ "다음 버전 곧 출시됩니다" (구체적 시기 확정 전 금지, 2월 '3월 출시' 약속 미이행 학습)
- ❌ "서버 문제" / "개발 중" / "구현 중" 류 핑계 (사용자에겐 의미 없음)
- ❌ 경쟁 앱 언급 / 비교
- ❌ "프리미엄 기능이라" / "업그레이드하시면" (무료 공개 메시지 훼손)
- ❌ 이모지 3개 이상 연속 사용
- ❌ 같은 답변 복붙 3회 이상 (아이디 식별 시 성의 없어 보임)

| T+24h | 저장된 유저 (UGC) 첫 노출 확인 → 리포스트 협의 검토 |
