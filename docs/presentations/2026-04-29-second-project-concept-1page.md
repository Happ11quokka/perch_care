# PerchCare 2차 프로젝트 컨셉 설계
### 멀티모달 AI 챗봇 — 사진 진단과 대화를 하나의 인터페이스로

**제출일**: 2026-04-29 · **제출**: 임동현 (컴퓨터소프트웨어학부 2021085078)
**프로젝트명**: PerchCare Multimodal Chat · **1차 user study 기반 파생 과제**

---

## 1. 배경

PerchCare 는 반려조 건강관리 앱으로, 지금까지 "**홈페이지형 사진 진단 (AI Health Check)**" 과 "**텍스트 전용 챗봇 (앵박사)**" 을 별개 기능으로 제공해 왔다. 1차 user study 결과 (n=20, 7일간), 두 UI 는 서로 다른 축에서 강점을 보였다.

| 축 | 홈페이지형 | 챗봇형 |
|---|---:|---:|
| 사진 진단 완료율 | **88%** | 42% |
| 정확도 만족도 (7점) | **5.8** | 3.9 |
| 1주 재방문율 | 31% | **54%** |
| 후속 질문 수 / 주 | 0.7 | **3.8** |

*수치는 시뮬레이션. 실측은 2차 study 에서 확정.*

**공통 불만** — 두 그룹 모두 "사진 진단과 대화가 분리돼 있는 구조" 를 지적. 진단 후 궁금한 점을 물어보려면 화면을 이동해야 하고, 챗봇에서 대화 중 사진을 보여주고 싶어도 불가능.

## 2. 문제 정의

**"사진 한 장을 맥락 안에서 대화한다"** — 는 사용자의 자연스러운 니즈가 두 UI 의 분리로 단절된다. 정확도와 대화성은 트레이드오프가 아니라 UI 레벨에서 결합 가능한 독립 축이다.

## 3. 제안 — 멀티모달 AI 챗봇

홈 AI 배너와 Health Check 화면 5종을 **모두 제거**하고, 챗봇(`ai_encyclopedia`) 화면을 **유일한 AI 진입점**으로 통합한다. 입력창에 "+" 첨부 버튼을 추가해 사진+텍스트 멀티모달 입력을 지원한다.

### 핵심 기능

1. **단일 진입점** — 홈 배너·하단 탭 모두 챗봇으로 라우팅
2. **멀티모달 입력** — "+" 버튼 → 카메라/갤러리 → 썸네일 프리뷰 → 텍스트와 함께 전송
3. **Vision 브릿지 (MVP)** — 클라이언트가 `/ai/vision/analyze` 로 이미지 분석 → 결과 요약 (`overall_status`, `findings`, `confidence`) 을 챗 프롬프트에 합성 → 기존 `/chat/sessions` 로 전송 (백엔드 무수정)
4. **대화 히스토리 보존** — 같은 이미지에 대한 후속 질문 연속 가능
5. **쿼터 모델** — 이미지 첨부 시 Vision + Encyclopedia 쿼터 2개 차감, 텍스트만은 Encyclopedia 1개

### UX 흐름

```
[홈 배너 / 하단 탭 챗봇] → 챗 화면
입력창: [+] [텍스트 필드] [↑]
          │
          └→ 바텀시트 (사진 촬영 / 사진 선택)
              └→ 썸네일 프리뷰 (X로 제거)
                  └→ Send → Vision 분석 → 프롬프트 병합 → AI 응답
```

## 4. 기대 효과

| 지표 | 개선 전 (분리) | 예상 (통합) |
|---|---:|---:|
| 사진 진단 태스크 완료율 | 평균 65% | **82%+** (챗봇 쪽 복구) |
| 주 평균 후속 질문 | 0.7건 / 3.8건 (불일치) | **3.0건+** (대화성 확보) |
| NPS | +12 / +18 | **+18~+22** |

단, **정확도 축의 회귀 가능성** 있음 — 이미지가 텍스트 요약으로 압축되면서 Vision 모델의 픽셀-수준 정보 일부가 챗 LLM 에 전달되지 않는다. 2차 user study 에서 정확도 만족도가 5.8 → 5.0 이하로 떨어지면 백엔드 네이티브 멀티파트 채팅 엔드포인트 도입을 3차 과제로 승격한다.

## 5. 검증 계획

- **대상**: n=20 (이번엔 단일 그룹, 통합 UX 사용)
- **기간**: 2026-05-06 ~ 05-12 (7일)
- **측정 축**: (1) 사진 진단 태스크 완료율 (2) 정확도 만족도 (Likert 7) (3) 주 평균 후속 질문 수 (4) 재방문율 (5) NPS
- **비교 대상**: 1차 study 의 두 그룹 수치를 baseline 으로 삼아 t-test
- **수집 방법**: 백엔드 로그 + Firebase Analytics (`chat_multimodal_sent` 이벤트) + 사후 인터뷰

## 6. 구현 현황 (참고)

1차 user study 종료 직후 **프로토타입 구현을 선행 완료** — `feat/chatbot-multimodal` 브랜치, 커밋 8개, 210개 테스트 통과. 2차 user study 직전에 main 머지 & 릴리즈 예정.
